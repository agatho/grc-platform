# ISMS Testplan — ISO 27001:2022 / ISO 27002:2022 / ISO 27005:2022

## Normenkonformer End-to-End Workflow

### Phase 1: Kontext & Scope (ISO 27001 Kap. 4)

- [ ] S1.1: ISMS-Übersicht aufrufen → Dashboard mit KPIs sichtbar
- [ ] S1.2: Assets anlegen (min. 3 verschiedene Tiers)
- [ ] S1.3: Asset-Owner zuweisen

### Phase 2: Risikobewertung (ISO 27001 Kap. 6.1.2 / ISO 27005)

- [ ] S2.1: Schutzbedarf festlegen (C/I/A für jedes Asset)
- [ ] S2.2: Bedrohungen identifizieren und zuordnen
- [ ] S2.3: Schwachstellen dokumentieren
- [ ] S2.4: ISMS-Bewertung erstellen (Assessment Run)
- [ ] S2.5: Risikoanalyse durchführen (Eintrittswahrscheinlichkeit × Auswirkung)
- [ ] S2.6: Risikobehandlung festlegen (Akzeptieren/Mitigieren/Transferieren/Vermeiden)

### Phase 3: Risikobehandlung (ISO 27001 Kap. 6.1.3)

- [ ] S3.1: Statement of Applicability (SoA) befüllen
- [ ] S3.2: Kontrollen zuweisen (aus ISO 27001 Annex A)
- [ ] S3.3: Kontrollstatus setzen (implementiert/geplant/nicht implementiert)
- [ ] S3.4: Begründung für nicht-anwendbare Kontrollen

### Phase 4: Implementierung & Betrieb (ISO 27001 Kap. 8)

- [ ] S4.1: Kontrollen maturity bewerten (0-5 Skala)
- [ ] S4.2: Evidenz zu Kontrollen hinzufügen
- [ ] S4.3: Sicherheitsvorfall melden und bearbeiten
- [ ] S4.4: Incident-Playbook zuweisen und durchlaufen

### Phase 5: Überwachung & Review (ISO 27001 Kap. 9)

- [ ] S5.1: Sicherheitslage (Posture) prüfen
- [ ] S5.2: Management Review erstellen und dokumentieren
- [ ] S5.3: Zertifizierungs-Readiness prüfen

### Phase 6: Verbesserung (ISO 27001 Kap. 10)

- [ ] S6.1: Feststellungen/Nichtkonformitäten dokumentieren
- [ ] S6.2: Korrekturmaßnahmen planen und nachverfolgen
- [ ] S6.3: Re-Assessment durchführen
