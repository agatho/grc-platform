# DPMS Testplan — DSGVO / BDSG / ISO 27701

## Normenkonformer End-to-End Workflow

### Phase 1: Verarbeitungsverzeichnis (Art. 30 DSGVO)

- [ ] D1.1: DPMS-Dashboard aufrufen (/dpms)
- [ ] D1.2: Neue Verarbeitungstätigkeit anlegen (/dpms/ropa)
- [ ] D1.3: Pflichtfelder: Zweck, Rechtsgrundlage, Betroffenengruppen, Datenkategorien
- [ ] D1.4: Empfänger und Drittlandtransfers dokumentieren
- [ ] D1.5: Löschfristen definieren
- [ ] D1.6: TOM-Referenz hinterlegen

### Phase 2: Datenschutz-Folgenabschätzung (Art. 35 DSGVO)

- [ ] D2.1: Neue DSFA erstellen (/dpms/dpia)
- [ ] D2.2: Schwellwertanalyse durchführen (DPIA-Kriterien)
- [ ] D2.3: Risiken für Betroffene bewerten
- [ ] D2.4: Maßnahmen zur Risikominderung definieren
- [ ] D2.5: Stellungnahme DSB dokumentieren

### Phase 3: Betroffenenrechte (Art. 15-22 DSGVO)

- [ ] D3.1: Betroffenenanfrage entgegennehmen (/dpms/dsr)
- [ ] D3.2: Anfragetyp klassifizieren (Auskunft, Löschung, Widerspruch etc.)
- [ ] D3.3: Frist überwachen (1 Monat / Art. 12 Abs. 3)
- [ ] D3.4: Anfrage bearbeiten und abschließen

### Phase 4: Datenpannenmanagement (Art. 33/34 DSGVO)

- [ ] D4.1: Datenpanne melden (/dpms/breaches)
- [ ] D4.2: 72h-Meldefrist an Aufsichtsbehörde tracken
- [ ] D4.3: Schweregrad bewerten (Risiko für Betroffene)
- [ ] D4.4: Benachrichtigung Betroffener dokumentieren (Art. 34)
- [ ] D4.5: Korrekturmaßnahmen dokumentieren

### Phase 5: Drittlandtransfer (Art. 44-49 DSGVO)

- [ ] D5.1: Transfer Impact Assessment erstellen (/dpms/tia)
- [ ] D5.2: Angemessenheitsbeschluss oder Garantien dokumentieren
- [ ] D5.3: SCCs/BCRs referenzieren

### Phase 6: Einwilligung & Aufbewahrung

- [ ] D6.1: Einwilligungsverwaltung prüfen (/dpms/consent)
- [ ] D6.2: Aufbewahrungsrichtlinien definieren (/dpms/retention)
