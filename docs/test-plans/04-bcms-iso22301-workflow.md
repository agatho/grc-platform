# BCMS Testplan — ISO 22301:2019

## Normenkonformer End-to-End Workflow

### Phase 1: Kontext & Scope (ISO 22301 Kap. 4)

- [ ] B1.1: BCMS-Dashboard aufrufen (/bcms)
- [ ] B1.2: Wesentliche Geschäftsprozesse identifizieren

### Phase 2: Business Impact Analyse (ISO 22301 Kap. 8.2.2)

- [ ] B2.1: BIA erstellen (/bcms/bia)
- [ ] B2.2: RTO/RPO für jeden Prozess definieren
- [ ] B2.3: Abhängigkeiten und Ressourcen dokumentieren
- [ ] B2.4: Maximale tolerierbare Ausfallzeit (MTPD) setzen

### Phase 3: Risikobeurteilung (ISO 22301 Kap. 8.2.3)

- [ ] B3.1: Krisenszenarien definieren (/bcms/crisis)
- [ ] B3.2: Eintrittswahrscheinlichkeit und Auswirkung bewerten
- [ ] B3.3: Szenarien priorisieren

### Phase 4: BC-Strategien (ISO 22301 Kap. 8.3)

- [ ] B4.1: Kontinuitätsstrategien definieren (/bcms/strategies)
- [ ] B4.2: Kosten der Strategien dokumentieren
- [ ] B4.3: Strategien den Szenarien zuordnen

### Phase 5: BC-Pläne (ISO 22301 Kap. 8.4)

- [ ] B5.1: Notfallplan erstellen (/bcms/plans)
- [ ] B5.2: Eskalationsprozeduren definieren
- [ ] B5.3: Kommunikationsplan integrieren
- [ ] B5.4: Plan an Szenarien und BIA verknüpfen

### Phase 6: Übungen & Tests (ISO 22301 Kap. 8.5)

- [ ] B6.1: BC-Übung planen (/bcms/exercises)
- [ ] B6.2: Übungsergebnis dokumentieren
- [ ] B6.3: Lessons Learned erfassen
- [ ] B6.4: Feststellungen aus Übung tracken

### Phase 7: Überwachung (ISO 22301 Kap. 9)

- [ ] B7.1: Resilienz-Dashboard prüfen (/bcms/resilience)
- [ ] B7.2: KPIs überwachen (BCP-Abdeckung, Übungshäufigkeit)
