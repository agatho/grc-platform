# ERM Testplan — ISO 31000:2018 / COSO ERM 2017

## Normenkonformer End-to-End Workflow

### Phase 1: Kontext etablieren (ISO 31000 Kap. 5.4)

- [ ] E1.1: Risikomanagement-Dashboard aufrufen (/risks)
- [ ] E1.2: Risikomethodik konfigurieren (/settings/risk-methodology)
- [ ] E1.3: Risikoappetit definieren (/erm/risk-appetite)
- [ ] E1.4: Risikokategorien und Bewertungskriterien prüfen

### Phase 2: Risikoidentifikation (ISO 31000 Kap. 6.4.2)

- [ ] E2.1: Neues Risiko erstellen (/risks/new)
- [ ] E2.2: Risikokategorie zuweisen (Cyber, Compliance, Operativ, Reputation)
- [ ] E2.3: Verantwortlichen (Risk Owner) zuweisen
- [ ] E2.4: Risiko aus Katalog erstellen (CatalogWorkqueue)
- [ ] E2.5: Risikogruppe zuweisen (/risks/group)

### Phase 3: Risikoanalyse (ISO 31000 Kap. 6.4.3)

- [ ] E3.1: Inhärente Bewertung durchführen (Eintrittswahrscheinlichkeit × Auswirkung)
- [ ] E3.2: Heatmap prüfen (/controls/heatmap)
- [ ] E3.3: FAIR-Analyse durchführen (/erm/fair)
- [ ] E3.4: Risikoindikatoren (KRI) definieren (/risks/kris)

### Phase 4: Risikobewertung (ISO 31000 Kap. 6.4.4)

- [ ] E4.1: Risiko-Score residual berechnen
- [ ] E4.2: Risikoappetit-Überschreitung prüfen
- [ ] E4.3: Priorisierung und Ranking

### Phase 5: Risikobehandlung (ISO 31000 Kap. 6.5)

- [ ] E5.1: Behandlungsoption wählen (Vermeiden, Mindern, Transferieren, Akzeptieren)
- [ ] E5.2: Kontrolle zuweisen
- [ ] E5.3: Maßnahmenplan erstellen
- [ ] E5.4: Budget zuweisen (/budget)
- [ ] E5.5: ROI/RONI berechnen (/budget/roi)

### Phase 6: Überwachung (ISO 31000 Kap. 6.6)

- [ ] E6.1: KRI-Monitoring (/risks/kris) — Schwellenwerte prüfen
- [ ] E6.2: RCSA-Kampagne erstellen (/rcsa)
- [ ] E6.3: Prädiktive Analyse prüfen (/predictive-risk)
- [ ] E6.4: Risiko-Re-Assessment durchführen
