Date: 2026-05-23
Scope: API routes (`apps/web/src/app/api/v1/**`), middleware, worker fetch

## Top 5 fixes by priority

1. **BI-Reports execute — RLS and read-only transaction are not applied** (`apps/web/src/app/api/v1/bi-reports/queries/execute/route.ts:59-66`). The `SET LOCAL app.current_org_id` runs in one `db.execute` call, the actual user query runs in a separate call → different implicit transaction → GUC and READ ONLY are dropped before the user query executes. Combined with the 402 tables without `FORCE ROW LEVEL SECURITY` (rbac-rls-audit), the schema-owner connection bypasses RLS, so admin/risk_manager can read across orgs from inside this endpoint. The advertised "RLS enforced" comment is false. Fix: run both statements inside `db.transaction(async (tx) => { ... })` so the GUC persists through the user query.
2. **Branding logo + favicon upload writes to caller-controlled path without org-membership check** (`apps/web/src/app/api/v1/organizations/[id]/branding/logo/route.ts:42-50`, `.../branding/favicon/route.ts:44-53`). The `orgId` comes from the URL, not the session, and is fed straight into `join(UPLOAD_DIR, orgId)`. There is no check that the caller's `ctx.orgId === params.id`. An admin of org A can overwrite the logo file of org B. Also accepts `image/svg+xml` and serves it from `/uploads/` — stored XSS on every page that renders the org logo. Fix: assert `id === ctx.orgId` (or admin-of-target-org), reject SVG (or sanitize + serve with `Content-Disposition: attachment`), and validate `orgId` against a UUID regex before using it in a filesystem path.
3. **XLIFF translation import concatenates `unit.entityId` into raw SQL** (`apps/web/src/app/api/v1/translations/import/route.ts:117-119, 144-146, 272-274, 297-300`). Only `field` and `tableName` are whitelisted; `entityId` arrives via `parseXliff` (no UUID validation, `unescapeXml` resurrects `&apos;` → `'`) and is concatenated into `WHERE id = '${unit.entityId}'`. Privileged (admin/risk_manager) but a tampered XLIFF file is a realistic attack surface. Fix: validate `entityId` with `z.string().uuid()` after parsing, or swap the raw SQL for parameterised `sql\`...${entityId}...\``.
4. **Break-glass admin login has no rate limit** (`apps/web/src/app/api/v1/auth/admin-login/route.ts:9-95`). With the known default `admin@arctos.dev / admin123` (see MEMORY) an attacker can brute-force unimpeded. fail2ban at Caddy only catches Auth.js routes by default, not this custom endpoint. Fix: wrap with the existing `rate-limit` helper, e.g. 5 attempts per IP per 15 min.
5. **Catalog-resolver, eam/dashboards/portfolio-optimization, translations/progress + heatmap interpolate strings into `sql.raw`** (`apps/web/src/lib/catalog-resolver.ts:80-91`, `apps/web/src/app/api/v1/translations/progress/route.ts:57-60`, `.../heatmap/route.ts:62-65`, `eam/dashboards/portfolio-optimization/route.ts:30-37`). Each site uses a whitelist OR a hard-coded dimensions array OR a manual quote-escape today, so none of them are exploitable. But the pattern keeps reappearing — one careless caller and the next reviewer flags it again. Fix: extract a `sqlIdent(name)` helper that wraps `sql.raw` only after a strict allow-list check, and migrate these sites to use it.

## Verification notes (false-positive triage)

- `apps/web/src/app/api/v1/dms/documents/route.ts`, `apps/web/src/app/api/v1/vendors/[id]/assessments/route.ts` — both are `export { ... } from "..."` aliases inheriting auth from the canonical route. Not bypasses.
- `apps/web/src/app/api/v1/admin/{users,api-keys,sso-providers,organizations}/route.ts` — pure 308 redirects with no DB touches. Safe.
- `apps/web/src/app/api/v1/{compliance,marketplace,programmes,reports,rcsa}/route.ts`, `bcms/crisis/dashboard/route.ts`, `isms/{management-reviews,nis2}/route.ts`, `dpms/transfer-impact-assessments/route.ts`, `identity/route.ts` — discovery-payload stubs returning 308/405/404 with no data exposure.
- `apps/web/src/app/api/v1/{calendar/ical/[token],portal/mailbox/[token],portal/dd/[token]}/...` — token-authed; tokens verified equal-eq against DB column with `length ≥ 32` enforcement.
